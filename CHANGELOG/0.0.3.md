## [0.0.3] - 2025-12-11

### 추가됨
- **이전 노드 결과 전달 시스템**: 노드 실행 시 이전 노드의 결과를 다음 노드로 전달하는 기능
  - `NodeExecutionRequest`에 `previous_node_result` 필드 추가
  - 클라이언트에서 이전 노드 결과를 저장하고 다음 노드 실행 시 전달
  - 서버에서 이전 노드 결과를 컨텍스트에 추가하여 조건 노드 등에서 사용 가능
- **조건 노드 기본값 자동 설정**: 조건 노드 생성 시 이전 노드의 출력을 기반으로 `field_path`와 `compare_value` 자동 설정
  - 이전 노드의 출력 스키마를 기반으로 예시 값 생성
  - `field_path`에 `output.{key}` 형식으로 자동 설정 (예: `output.wait_time`)
  - `compare_value`에 해당 값 자동 설정
- **실행 결과 모달**: 스크립트/노드 실행 결과를 가운데 팝업 모달로 표시
  - 실행 요약 (성공/실패/중단 개수) 표시
  - 개별 노드/스크립트 실행 결과 상세 표시
  - 다크/라이트 모드 지원
  - 사이드바를 제외한 메인 콘텐츠 영역 중앙 정렬
- **노드 실행 로그 개선**: 노드 실행 시 전체 노드 개수와 현재 순번 표시
  - 로그 형식: `노드 {현재순번}/{전체개수} 실행 시작/성공/실패`
  - 시작/종료 노드도 전체 개수에 포함
- **비동기 로깅**: 노드 실행 로그를 비동기로 전송하여 실행 속도 개선
  - `asyncio.create_task()`를 사용한 fire-and-forget 로깅
  - 로깅이 노드 실행을 블로킹하지 않음

### 변경됨
- **조건 노드 로직 분리**: 조건 노드 전처리 로직을 `ConditionService`로 분리
  - `server/services/condition_service.py` 파일 생성
  - 조건 노드 실행 전 이전 노드 출력을 `previous_output`에 주입
- **파라미터 병합 로직 개선**: `parameters` 필드의 파라미터를 `node_data`에 병합
  - DB에서 불러온 노드의 `parameters` 필드를 `node_data`에 병합
  - `parameters`가 우선순위가 높음
- **실행 결과 표시 방식 변경**: 오른쪽 사이드바에서 가운데 팝업 모달로 변경
  - `result-modal.js` 파일 생성 (모달 전용 관리)
  - `modal.js`에서 실행 결과 관련 코드 제거
- **CSS 구조 개선**: 모달별, 테마별로 CSS 파일 분리
  - `components/modal.css`, `components/node-settings-modal.css`, `components/result-modal.css` (구조)
  - `themes/dark/modal.css`, `themes/light/modal.css` (테마별)
  - 각 CSS 속성에 상세 주석 추가
- **레거시 코드 제거**: 하위 호환성을 위한 레거시 코드 제거
  - 파라미터 처리를 `parameters` 필드로 통일
  - 필수 파라미터가 없으면 에러 발생

### 수정됨
- **대기 노드 대기 시간 적용 문제**: UI에서 변경한 대기 시간이 실행에 반영되지 않던 문제 수정
  - `parameters` 필드의 파라미터를 `node_data`에 병합하도록 수정
- **조건 노드에 이전 노드 출력 전달 문제**: 조건 노드 실행 시 이전 노드의 출력이 전달되지 않던 문제 수정
  - 클라이언트에서 이전 노드 결과를 저장하고 다음 노드 실행 시 전달
  - 서버에서 이전 노드 결과를 컨텍스트에 추가
- **노드 실행 로그 형식**: 모든 노드가 "1/1"로 표시되던 문제 수정
  - 전체 노드 개수와 현재 순번을 API 요청에 포함
  - 시작/종료 노드도 전체 개수에 포함
- **대기 노드 실행 속도**: 로깅이 노드 실행을 블로킹하여 느려 보이던 문제 수정
  - 비동기 로깅으로 변경하여 실행 속도 개선
- **조건 노드 파라미터 누락 에러**: 조건 노드에 필수 파라미터가 없을 때 에러 발생하던 문제 수정
  - 기본값이 있는 파라미터는 필수 파라미터 검증에서 제외
