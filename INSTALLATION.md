# 설치 및 실행 가이드

AutoScript를 설치하고 실행하는 방법을 안내합니다.

## 시스템 요구사항

- **Python**: 3.10.6 이상
- **운영체제**: Windows 10/11
- **메모리**: 최소 4GB RAM 권장
- **디스크**: 최소 1GB 권장

## 설치 및 실행

### 1. 저장소 클론

```bash
git clone <repository-url>
cd AutoScript
```

### 2. 가상환경 생성 및 활성화

> **중요**: 가상환경 이름은 반드시 `venv`로 생성해야 합니다. 다른 이름을 사용하면 자동 스크립트가 가상환경을 찾지 못할 수 있습니다.

```bash
python -m venv venv
venv\Scripts\activate
```

### 3. 패키지 설치

```bash
cd server
pip install -r requirements.txt
```

### 4. 환경 변수 설정 (선택사항)

프로젝트 루트 디렉토리에 `.env` 파일을 생성하여 서버 설정을 변경할 수 있습니다:

```bash
# 프로젝트 루트 디렉토리에서 .env 파일 생성
```

`.env` 파일 예시:

```env
# API 설정
API_HOST=127.0.0.1  # 보안: 로컬호스트에서만 접근 가능 (기본값)
API_PORT=8001  # 서버 포트 (기본값: 8001)

# 환경 설정
ENVIRONMENT=prd  # prd: 프로덕션 모드, dev: 개발 모드
```

> **참고**: 
> - `.env` 파일이 없어도 기본값으로 서버가 실행됩니다 (호스트: 127.0.0.1, 포트: 8001)
> - `.env` 파일을 생성하면 `start-server.bat`가 자동으로 설정을 읽어서 사용합니다
> - 자세한 설정 옵션은 [환경 변수 설정 가이드](docs/dev/environment.md)를 참고하세요

### 5. 서버 실행

서버를 실행하는 방법은 두 가지가 있습니다:

#### 방법 1: 배치 파일 사용 (권장) ⚡

프로젝트 루트에서 `start-server.bat` 파일을 더블클릭하거나 명령어로 실행합니다:

```bash
start-server.bat
```

**배치 파일 기능:**
- 가상환경 자동 활성화 (`venv` 폴더 자동 감지)
- Python 설치 여부 자동 확인
- 서버 자동 실행 (호스트: 127.0.0.1, 포트: 8001)

> **참고**: 가상환경 이름은 반드시 `venv`로 생성해야 합니다. `start-server.bat`가 `venv` 폴더를 우선적으로 찾습니다.

> **보안 참고**: `127.0.0.1`은 로컬호스트에서만 접근 가능하며, 로컬 네트워크의 다른 기기에서는 접근할 수 없습니다.

#### 방법 2: 수동 실행

```bash
cd server
python -m uvicorn main:app --reload --host 127.0.0.1 --port 8001
```

> **참고**: 
> - 기본 포트는 8001입니다. 포트를 변경하려면 `start-server.bat` 파일을 수정하거나 수동 실행 시 `--port` 옵션을 변경하세요.
> - `--host 127.0.0.1` 옵션은 로컬호스트에서만 접근 가능하도록 설정합니다.

### 6. 웹 브라우저에서 접속

서버가 실행되면 브라우저에서 다음 주소로 접속하세요:

- `http://localhost:8001`
- `http://127.0.0.1:8001`
