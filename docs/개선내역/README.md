# 개선내역 문서

이 폴더에는 프로젝트 개발 과정에서 해결한 주요 문제들과 해결 방법을 문서화한 내용이 담겨 있습니다.

## 문서 목록

### 백엔드/인프라 관련

### 1. [환경 변수 중앙 관리 시스템 구축](./환경-변수-중앙-관리-시스템.md)
- **문제**: 여러 파일에서 환경 변수를 개별적으로 읽어 중복과 불일치 발생
- **해결**: `config/server_config.py`를 통한 중앙 관리 시스템 구축
- **결과**: 단일 설정값 공급원, 코드 재사용성 향상, 유지보수성 개선

### 2. [로그 시스템 개선: 동적 디렉토리 설정 및 레벨 제어](./로그-시스템-개선.md)
- **문제**: 하드코딩된 로그 경로, 설정 미적용, 싱글톤 패턴 버그
- **해결**: `LOG_DIR` 환경 변수 사용, 로그 레벨 적용, 싱글톤 패턴 수정
- **결과**: 유연한 로그 경로 설정, 환경별 로그 레벨 제어

### 3. [파일명 충돌 해결: config.py vs config/ 패키지](./파일명-충돌-해결.md)
- **문제**: `server/config.py`와 `server/config/` 패키지의 이름 충돌
- **해결**: `config/server_config.py`로 이동
- **결과**: Python import 시스템 정상 작동, 명확한 파일 구조

### 4. [UTF-8 BOM 문제 해결: 환경 변수 읽기 실패](./UTF-8-BOM-문제-해결.md)
- **문제**: `.env` 파일의 UTF-8 BOM으로 인한 환경 변수 읽기 실패
- **해결**: UTF-8 without BOM으로 파일 재저장
- **결과**: 환경 변수 정상 읽기, 설정 올바르게 적용

## 공통 주제

모든 문서에서 공통적으로 다루는 주제:

1. **중앙화**: 설정과 로직의 중앙 관리
2. **일관성**: 프로젝트 전반의 일관된 패턴 적용
3. **유지보수성**: 코드 변경 시 영향 범위 최소화
4. **확장성**: 향후 기능 추가를 고려한 설계

## 기술 스택

- **Python 3.10+**: 백엔드 서버
- **FastAPI**: 웹 프레임워크
- **python-dotenv**: 환경 변수 관리
- **SQLite**: 데이터베이스
- **JavaScript (ES6+)**: 프론트엔드

## 개발 원칙

이 프로젝트에서 적용한 개발 원칙:

1. **DRY (Don't Repeat Yourself)**: 코드 중복 최소화
2. **Single Source of Truth**: 단일 진실 공급원
3. **Separation of Concerns**: 관심사의 분리
4. **Configuration over Code**: 코드보다 설정 우선
