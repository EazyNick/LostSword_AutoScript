# 파일명 충돌 해결: config.py vs config/ 패키지

## 문제 상황

프로젝트에 `server/config.py` 파일과 `server/config/` 디렉토리가 동시에 존재하여 Python import 시스템에서 충돌이 발생했습니다.

**에러 메시지:**
```
ImportError: cannot import name 'settings' from 'config' 
(C:\Users\User\Desktop\python\LostSword_AutoScript\server\config\__init__.py)
```

## 원인

Python의 import 시스템은 다음과 같이 동작합니다:

1. `from config import settings` 호출 시
2. Python은 먼저 `config` 패키지(디렉토리)를 찾음
3. `server/config/__init__.py`를 로드
4. `server/config.py` 파일은 무시됨

따라서 `server/config.py`의 `settings` 객체를 import할 수 없었습니다.

## 해결 방법

`server/config.py`를 `server/server_config.py`로 파일명을 변경하여 패키지와의 충돌을 해결했습니다.

```bash
# 파일명 변경
server/config.py → server/server_config.py
```

## 결과 및 장점

### 1. 명확한 구분

- 설정 파일(`server_config.py`)과 설정 패키지(`config/`)가 명확히 구분됨
- 파일 구조가 더 직관적이고 이해하기 쉬움

### 2. Python import 시스템 정상 작동

- import 충돌이 해결되어 모든 모듈이 정상적으로 로드됨
- 코드 실행 시 에러 없이 동작

### 3. 유지보수성 향상

- 파일명이 역할을 명확히 표현 (`server_config` = 서버 설정)
- 새로운 개발자도 파일 구조를 쉽게 이해 가능

### 4. 확장성

- 향후 `config/` 패키지에 새로운 설정 모듈을 추가해도 충돌 없음
- 각각의 역할이 명확히 분리되어 있음

## 교훈

Python 프로젝트에서 파일명을 지을 때 주의사항:

1. **모듈과 패키지 이름 충돌**: 같은 이름의 `.py` 파일과 디렉토리를 함께 사용하지 말 것
2. **명확한 네이밍**: 파일명이 역할을 명확히 표현할 것
3. **일관성**: 프로젝트 전체에서 일관된 네이밍 규칙 사용
