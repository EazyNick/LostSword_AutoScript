# UI/UX 설계 포트폴리오

이 폴더에는 사용자 인터페이스와 사용자 경험 설계 과정에서 내린 결정들과 그 이유를 문서화한 내용이 담겨 있습니다.

## 문서 목록

### 1. [노드 기반 워크플로우 편집기](./노드-기반-워크플로우-편집기.md)
- **문제**: 텍스트 기반 스크립트 작성의 한계
- **해결**: Figma 스타일의 노드 기반 시각적 편집기
- **결과**: 직관적인 인터페이스, 시각적 검증, 확장성

### 2. [사이드바 설계](./사이드바-설계.md)
- **문제**: 여러 스크립트 관리 및 전환의 어려움
- **해결**: 리사이즈 가능한 왼쪽 사이드바, 드래그 앤 드롭 순서 변경
- **결과**: 효율적인 스크립트 관리, 직관적인 인터페이스

### 3. [헤더 액션 버튼 설계](./헤더-액션-버튼-설계.md)
- **문제**: 자주 사용하는 액션에 빠르게 접근하기 어려움
- **해결**: 색상 구분, 아이콘+텍스트, 실행 중 상태 표시
- **결과**: 빠른 액션 접근, 명확한 상태 표시

### 4. [모달 및 알림 시스템](./모달-및-알림-시스템.md)
- **문제**: 일관성 없는 알림 방식, 접근성 부족
- **해결**: 통합 모달 시스템, Toast 알림, 키보드 지원
- **결과**: 일관된 사용자 경험, 명확한 피드백

### 5. [반응형 디자인](./반응형-디자인.md)
- **문제**: 다양한 화면 크기와 디바이스 지원 부족
- **해결**: 모바일 퍼스트 접근, 브레이크포인트 설정, 터치 제스처
- **결과**: 다양한 디바이스 지원, 일관된 경험

## 공통 설계 원칙

모든 UI/UX 설계에서 공통적으로 적용한 원칙:

### 1. 사용자 중심 설계
- 사용자의 작업 흐름을 분석하여 인터페이스 설계
- 직관적이고 학습 곡선이 낮은 인터페이스

### 2. 일관성
- 모든 화면에서 동일한 패턴과 스타일 사용
- 예측 가능한 인터랙션 제공

### 3. 접근성
- 키보드만으로도 모든 기능 사용 가능
- 스크린 리더 지원
- 충분한 색상 대비

### 4. 피드백
- 모든 액션에 즉각적인 시각적 피드백
- 명확한 상태 표시 (로딩, 성공, 실패)

### 5. 성능
- 부드러운 애니메이션 (60fps)
- GPU 가속 활용
- 불필요한 리플로우 방지

## 디자인 시스템

### 색상 팔레트

**주요 색상:**
- **파란색** (#3182ce): 주요 액션, 링크
- **초록색** (#38a169): 성공, 실행
- **주황색** (#ed8936): 경고, 전체 실행
- **빨간색** (#ef4444): 에러, 삭제
- **회색** (#4a5568): 보조 액션, 비활성

**배경:**
- **다크 그레이** (#1a202c): 메인 배경
- **미디엄 그레이** (#2d3748): 카드 배경
- **라이트 그레이** (#4a5568): 경계선

### 타이포그래피

**폰트 크기:**
- **제목**: 24px (헤더), 18px (사이드바)
- **본문**: 14px
- **작은 텍스트**: 12px

**폰트 굵기:**
- **제목**: 700 (bold)
- **부제목**: 600 (semibold)
- **본문**: 400 (normal)

### 간격 시스템

**기준 단위:** 4px

**사용 예시:**
- 작은 간격: 4px, 8px
- 중간 간격: 12px, 16px
- 큰 간격: 20px, 24px, 32px

### 애니메이션

**지속 시간:**
- 빠른 전환: 0.2s
- 일반 전환: 0.3s
- 느린 전환: 0.5s

**이징:**
- 기본: `ease`
- 부드러운: `ease-in-out`
- 자연스러운: `cubic-bezier(0.4, 0, 0.2, 1)`

## 사용자 테스트 결과

### 긍정적 피드백

1. **직관성**: "코드 없이도 복잡한 워크플로우를 만들 수 있어요"
2. **시각적 피드백**: "실행 중일 때 애니메이션이 있어서 좋아요"
3. **일관성**: "모든 화면에서 동일한 방식으로 작동해요"
4. **접근성**: "키보드만으로도 사용할 수 있어요"

### 개선 필요 사항

1. **학습 자료**: 초보자를 위한 튜토리얼 필요
2. **키보드 단축키**: 더 많은 단축키 추가 요청
3. **다크 모드**: 시스템 설정 연동 요청
4. **모바일 최적화**: 모바일에서 일부 기능 제한

## 향후 개선 방향

1. **접근성 강화**: WCAG 2.1 AA 수준 달성
2. **다국어 지원**: 영어, 일본어 등 추가 언어 지원
3. **테마 시스템**: 사용자 정의 테마 지원
4. **애니메이션 커스터마이징**: 사용자가 애니메이션 속도 조절 가능
