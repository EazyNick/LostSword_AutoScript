# 사이드바 설계: 스크립트 관리 인터페이스

## 문제 상황

여러 개의 자동화 스크립트를 관리하고 전환하기 위해서는 효율적인 인터페이스가 필요했습니다. 초기에는 다음과 같은 문제가 있었습니다:

1. **스크립트 전환 어려움**: 여러 스크립트 간 전환이 번거로움
2. **실행 순서 관리 부재**: 스크립트 실행 순서를 변경할 방법 없음
3. **스크립트 정보 부족**: 각 스크립트의 목적과 수정 날짜를 확인하기 어려움
4. **공간 효율성**: 작은 화면에서도 효율적으로 사용 가능해야 함

## 설계 결정

### 왼쪽 사이드바 배치

**레이아웃:**
```
┌─────────┬──────────────────┐
│ 프로필  │   헤더            │
├─────────┼──────────────────┤
│ 사이드바 │   메인 컨텐츠     │
│ (네비)  │   (페이지 영역)   │
│         │                  │
└─────────┴──────────────────┘
```

**구성 요소:**
- **상단 프로필 영역**: 사용자 프로필 및 닉네임 (좌측 최상단)
- **사이드바**: 네비게이션 메뉴 (대시보드, 스크립트, 실행 기록, 설정)
- **스크립트 목록**: 스크립트 페이지에서만 표시되는 스크립트 목록

**이유:**
1. **일관성**: 대부분의 IDE와 디자인 도구가 왼쪽 사이드바 사용
2. **접근성**: 왼쪽에 배치하여 키보드 단축키와 조합하기 쉬움
3. **공간 효율**: 세로 공간을 효율적으로 활용

### 스크립트 카드 디자인

**구성 요소:**
- **아이콘**: 스크립트 타입을 나타내는 시각적 표시 (📄)
- **제목**: 스크립트 이름
- **설명**: 스크립트의 목적을 간단히 설명
- **날짜**: 마지막 수정 날짜

**이유:**
1. **정보 밀도**: 한 눈에 필요한 정보를 모두 확인 가능
2. **시각적 구분**: 각 스크립트를 카드로 구분하여 선택하기 쉬움
3. **컨텍스트 제공**: 설명과 날짜로 스크립트의 용도를 파악 가능

### 리사이즈 가능한 사이드바

**구현:**
```css
.sidebar {
    width: 350px;
    min-width: 250px;
    max-width: 600px;
}

.sidebar-resize-handle {
    position: absolute;
    right: -4px;
    width: 8px;
    cursor: col-resize;
}
```

**이유:**
1. **사용자 맞춤**: 사용자가 원하는 너비로 조절 가능
2. **작업 공간 최적화**: 워크플로우 편집 시 더 넓은 공간 필요 시 축소
3. **다양한 화면 크기 지원**: 작은 화면에서도 사용 가능

### 드래그 앤 드롭으로 순서 변경

**구현:**
- 스크립트 항목을 드래그하여 순서 변경
- 드래그 중 시각적 피드백 (반투명, 경계선 표시)
- 드롭 위치 표시 (상단/하단)

**이유:**
1. **직관성**: 마우스로 드래그하여 순서 변경이 가장 직관적
2. **빠른 작업**: 여러 스크립트의 순서를 빠르게 재배치
3. **시각적 피드백**: 드래그 중 위치를 명확히 표시

## 기술적 구현

### 활성 스크립트 표시

```css
.script-item.active {
    border-color: #3182ce;
    background-color: #2d3748;
}
```

**이유:**
- 현재 편집 중인 스크립트를 명확히 구분
- 파란색 테두리로 시각적 강조

### 호버 효과

```css
.script-item:hover {
    background-color: #2d3748;
    transform: translateY(-2px);
    box-shadow: 0 5px 15px rgba(0, 0, 0, 0.3);
}
```

**이유:**
1. **인터랙션 피드백**: 마우스 오버 시 즉각적인 시각적 피드백
2. **깊이감**: 살짝 떠오르는 효과로 클릭 가능함을 암시
3. **전문성**: 세련된 UI/UX 제공

### 삭제 버튼 (호버 시 표시)

```css
.script-delete-btn {
    opacity: 0;
    transition: all 0.2s ease;
}

.script-item:hover .script-delete-btn {
    opacity: 1;
}
```

**이유:**
1. **깔끔한 UI**: 평소에는 숨겨져 있어 깔끔한 인터페이스 유지
2. **실수 방지**: 호버 시에만 표시하여 실수로 삭제하는 것 방지
3. **접근성**: 필요할 때 쉽게 접근 가능

### 스크립트 추가 버튼

**위치:** 사이드바 헤더 오른쪽

**이유:**
1. **명확한 위치**: 헤더에 배치하여 항상 접근 가능
2. **시각적 강조**: 파란색 배경으로 중요한 액션임을 표시
3. **일관성**: 다른 추가 버튼들과 동일한 스타일

## 사용자 경험 개선

### 스크립트 선택 시 즉시 전환

- 클릭 시 즉시 워크플로우 편집 영역이 해당 스크립트로 전환
- 로딩 중 시각적 피드백 제공

### 스크립트 정보 표시

- **제목**: 스크립트의 주요 목적
- **설명**: 상세한 설명 (선택사항)
- **날짜**: 마지막 수정 시간으로 최신성 파악

### 반응형 디자인

```css
@media (max-width: 768px) {
    .sidebar {
        width: 100%;
        height: 200px;
    }
}
```

**이유:**
- 모바일 환경에서도 사용 가능
- 작은 화면에서는 세로로 스크롤 가능한 목록으로 표시

## 결과

### 장점

1. **효율적인 스크립트 관리**: 여러 스크립트를 쉽게 전환하고 관리
2. **직관적인 인터페이스**: 드래그 앤 드롭으로 순서 변경
3. **공간 효율성**: 리사이즈 가능하여 작업 공간 최적화
4. **정보 제공**: 각 스크립트의 목적과 수정 날짜 확인 가능

### 사용자 피드백

- "여러 스크립트를 쉽게 관리할 수 있어요"
- "드래그로 순서를 바꾸는 게 편해요"
- "사이드바 크기를 조절할 수 있어서 좋아요"

## 향후 개선 방향

1. **스크립트 검색**: 많은 스크립트 중 빠르게 찾기
2. **스크립트 그룹화**: 카테고리별로 그룹화하여 관리
3. **즐겨찾기**: 자주 사용하는 스크립트를 상단에 고정
4. **최근 사용**: 최근에 편집한 스크립트 목록 표시
