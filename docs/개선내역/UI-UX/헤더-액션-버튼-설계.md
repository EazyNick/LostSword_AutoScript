# 헤더 액션 버튼 설계: 빠른 액션 접근

## 문제 상황

워크플로우 편집 중 자주 사용하는 액션(저장, 실행, 노드 추가 등)에 빠르게 접근할 수 있는 인터페이스가 필요했습니다. 초기에는 다음과 같은 문제가 있었습니다:

1. **액션 접근 어려움**: 메뉴를 여러 번 클릭해야 액션 실행
2. **상태 표시 부족**: 실행 중인지, 저장되었는지 확인 어려움
3. **시각적 피드백 부족**: 버튼 클릭 후 즉각적인 피드백 없음
4. **키보드 단축키 부재**: 빠른 작업을 위한 단축키 없음

## 설계 결정

### 상단 고정 헤더

**레이아웃:**
```
┌─────────────────────────────────────┐
│ [아이콘] 제목    [버튼] [버튼] ... │ ← 항상 상단에 고정
├─────────────────────────────────────┤
│                                     │
│      워크플로우 편집 영역            │
│                                     │
└─────────────────────────────────────┘
```

**이유:**
1. **항상 접근 가능**: 스크롤해도 항상 보이는 위치
2. **일관성**: 대부분의 애플리케이션이 상단 헤더 사용
3. **공간 효율**: 세로 공간을 최소한으로 사용

### 버튼 그룹화 및 색상 구분

**버튼 배치:**
- **왼쪽**: 문서 정보 (아이콘, 제목)
- **중앙**: 현재 스크립트 정보 (제목, 설명)
- **오른쪽**: 액션 버튼들

**색상 체계:**
- **파란색** (#3182ce): 추가/생성 액션 (노드 추가)
- **회색** (#4a5568): 저장 액션
- **초록색** (#38a169): 실행 액션
- **주황색** (#ed8936): 전체 실행 액션

**이유:**
1. **시각적 구분**: 색상으로 액션의 성격을 즉시 파악
2. **일관성**: 동일한 액션은 동일한 색상 사용
3. **직관성**: 초록색 = 실행, 파란색 = 추가 등 직관적

### 아이콘 + 텍스트 조합

**구성:**
```html
<button class="header-btn save-btn">
    <span class="btn-icon">💾</span>
    <span class="btn-text">저장</span>
</button>
```

**이유:**
1. **명확성**: 아이콘만으로는 모호할 수 있어 텍스트 추가
2. **접근성**: 스크린 리더 사용자도 이해 가능
3. **학습 곡선**: 초보자도 쉽게 이해

## 기술적 구현

### 실행 중 상태 표시

```css
.header-btn.executing {
    animation: pulse-executing 1.5s ease-in-out infinite;
}

.header-btn.executing::after {
    content: '';
    border: 2px solid rgba(255, 255, 255, 0.3);
    border-top-color: #ffffff;
    animation: spin 0.8s linear infinite;
}
```

**이유:**
1. **명확한 피드백**: 실행 중임을 시각적으로 명확히 표시
2. **사용자 안심**: 작업이 진행 중임을 알려 불안감 해소
3. **중복 실행 방지**: 실행 중에는 버튼 비활성화로 중복 실행 방지

### 호버 효과

```css
.header-btn:hover {
    transform: translateY(-2px);
    box-shadow: 0 4px 8px rgba(0, 0, 0, 0.2);
}
```

**이유:**
1. **인터랙션 피드백**: 마우스 오버 시 즉각적인 시각적 피드백
2. **클릭 가능성 암시**: 버튼이 클릭 가능함을 시각적으로 표현
3. **전문성**: 세련된 UI/UX 제공

### 그라데이션 배경

```css
.top-header {
    background: linear-gradient(135deg, #2d3748 0%, #1a202c 100%);
}
```

**이유:**
1. **시각적 깊이**: 평면적인 디자인보다 입체감 있는 디자인
2. **브랜드 일관성**: 어두운 테마와 조화
3. **전문성**: 고급스러운 느낌

## 사용자 경험 개선

### 버튼 그룹화

**그룹 1: 전체 실행**
- 전체 스크립트 실행 버튼 (주황색, 큰 버튼)

**그룹 2: 현재 스크립트 액션**
- 저장 (회색)
- 노드 추가 (파란색)
- 실행 (초록색)

**이유:**
1. **논리적 그룹화**: 관련된 액션을 함께 배치
2. **시각적 계층**: 중요한 액션(전체 실행)을 강조
3. **사용 패턴 반영**: 자주 사용하는 순서대로 배치

### 반응형 디자인

```css
@media (max-width: 768px) {
    .btn-text {
        display: none;  /* 작은 화면에서는 아이콘만 표시 */
    }
}
```

**이유:**
1. **공간 절약**: 작은 화면에서 텍스트 숨김으로 공간 확보
2. **가독성 유지**: 아이콘만으로도 액션 파악 가능
3. **일관성**: 모바일 환경에서도 동일한 기능 제공

### 툴팁 제공

```html
<button title="모든 스크립트 실행">...</button>
```

**이유:**
- 아이콘만 표시될 때도 기능을 설명
- 마우스 오버 시 추가 정보 제공

## 결과

### 장점

1. **빠른 액션 접근**: 자주 사용하는 액션에 즉시 접근 가능
2. **명확한 상태 표시**: 실행 중, 저장 완료 등 상태를 명확히 표시
3. **직관적인 인터페이스**: 색상과 아이콘으로 액션을 쉽게 파악
4. **일관성**: 모든 화면에서 동일한 헤더 사용

### 사용자 피드백

- "저장 버튼이 항상 보여서 편해요"
- "실행 중일 때 애니메이션이 있어서 좋아요"
- "색상으로 버튼을 구분할 수 있어서 직관적이에요"

## 향후 개선 방향

1. **키보드 단축키**: Ctrl+S (저장), F5 (실행) 등
2. **컨텍스트 메뉴**: 우클릭으로 추가 옵션 제공
3. **사용자 맞춤**: 버튼 순서와 표시 여부를 사용자가 설정
4. **알림 통합**: 저장 완료, 실행 완료 등의 알림을 헤더에 표시
