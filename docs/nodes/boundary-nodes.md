# 경계 노드 (Boundary Nodes)

경계 노드는 워크플로우의 시작점이나 종료점을 나타내는 특수한 노드입니다.

## 구현된 노드

### start (시작 노드)

워크플로우의 시작점을 나타내는 노드입니다.

**파일 위치**: `server/nodes/boundarynodes/start.py`

**노드 타입**: `start`

**설명**: 워크플로우 실행이 시작될 때 자동으로 실행되며, 다음 노드에 워크플로우 시작 신호를 전달합니다.

#### 파라미터

이 노드는 파라미터를 받지 않습니다.

#### 출력 스키마

```json
{
  "action": "start",
  "status": "completed",
  "output": {
    "time": "2025-12-21 12:34:56",
    "message": "워크플로우가 시작되었습니다."
  }
}
```

#### 동작 방식

1. 현재 시간을 한국 시간으로 가져옵니다 (`get_korea_time_str()`)
2. 시작 로그를 기록합니다
3. 표준 형식의 결과를 반환합니다

#### 코드 예시

```python
@NodeExecutor("start")
async def execute(parameters: dict[str, Any]) -> dict[str, Any]:
    time_str = get_korea_time_str()
    logger.info(f"[StartNode] 워크플로우 시작 - 시작 시간: {time_str}")
    
    return {
        "action": "start",
        "status": "completed",
        "output": {
            "time": time_str,
            "message": "워크플로우가 시작되었습니다."
        },
    }
```

#### 사용 예시

시작 노드는 일반적으로 워크플로우의 첫 번째 노드로 자동 추가됩니다. 사용자가 명시적으로 추가할 필요는 없습니다.

```
[시작] → [대기] → [클릭] → ...
```

## 특징

1. **자동 실행**: 워크플로우 실행 시 자동으로 첫 번째로 실행됩니다
2. **파라미터 불필요**: 별도의 설정이 필요 없습니다
3. **표준 출력**: 모든 노드와 동일한 표준 형식의 결과를 반환합니다
4. **시간 기록**: 워크플로우 시작 시간을 기록하여 추적 가능합니다
5. **동적 인식**: `isBoundary: true` 속성으로 경계 노드가 자동 인식되며, 하드코딩 없이 확장 가능합니다

## 기술적 세부사항

### 경계 노드 시스템 (v0.0.6+)

경계 노드는 `nodes_config.py`에서 `isBoundary: true` 속성으로 정의됩니다:

```python
"start": {
    "label": "시작 노드",
    "isBoundary": True,  # 경계 노드로 표시
    ...
}
```

시스템은 다음과 같이 경계 노드를 처리합니다:

1. **동적 인식**: `isBoundaryNodeSync()` 함수로 경계 노드 타입을 캐시하여 빠르게 확인
2. **자동 시작점**: 워크플로우 실행 시 경계 노드를 자동으로 찾아 시작점으로 사용
3. **확장성**: 새로운 경계 노드 추가 시 코드 수정 없이 자동 인식
